# IT 资产转借操作指南

## 1. 概述

本文档说明如何通过智能助手将已借用的 IT 资产转借给其他同事。转借功能允许借用人在借用期间将资产使用权临时转移给他人，无需归还后再重新借用。

---

## 2. 转借流程

### 2.1 发起转借

如你已借用某资产但暂时不再需要使用，可将资产转借给其他同事。发起转借需要提供：

| 信息 | 说明 | 示例 |
|------|------|------|
| **借用记录 ID** | 待转借的借用记录编号 | 5 |
| **接受人用户 ID** | 接收资产的同事用户编号 | user_002 |

**话术示例**：
- "将借用记录 5 转借给 user_002"
- "帮我把笔记本转给张三，借用 ID 是 5"
- "发起转借，borrowId=5，toUserId=user_002"

### 2.2 确认接收转借

接收人会收到转借请求，确认后转借正式生效：

**话术示例**：
- "确认接收转借 3"
- "同意转借 transferId=3"

> **注意**：转借确认操作必须由接收人本人执行，不可代操作。

### 2.3 查询转借状态

可通过个人借用记录查看转借是否已完成。

---

## 3. 转借规则

### 3.1 转借条件
- 仅允许对**状态为 `pending`（借用中）** 的记录发起转借
- 已取消(`cancelled`) 或已完成的借用记录不可转借
- 接收人必须在系统中存在有效用户身份

### 3.2 责任转移
- 转借确认后，资产使用权转移至接收人
- 原始借用人对该资产的保管责任随之转移
- 原始借用人不再对该资产的损坏、丢失承担责任

### 3.3 转借限制
- 同一资产不可同时转借给多人
- 转借后接收人可继续转借给第三方（需重新发起转借流程）
- 转借不改变资产的预计归还时间

---

## 4. 注意事项

### 4.1 操作前确认
- 发起转借前请与接收人沟通确认，避免误操作
- 确认接收人用户 ID 正确无误
- 转借一旦确认接收，不可撤销

### 4.2 异常处理
| 问题 | 解决方法 |
|------|---------|
| 找不到借用记录 | 确认借用记录 ID 是否正确，通过"查询我的借用记录"获取准确 ID |
| 接收人不存在 | 确认对方用户 ID 是否正确 |
| 转借确认失败 | 确认你本人是转借记录的接收人 |

### 4.3 安全提醒
- 不要将资产转借给未经授权的第三方
- 转借操作建议在内部沟通确认后再进行
- 如发现异常转借记录，立即联系 IT 部门处理

---

## 5. 常见问题

**Q: 转借后谁承担资产损坏责任？**
A: 转借确认接收后，责任自动转移到接收人。

**Q: 转借后还能取消吗？**
A: 转借在接收人确认前可以取消（通过取消原借用），确认接收后不可撤销。

**Q: 转借会影响归还日期吗？**
A: 不影响，归还日期以原始借用记录为准。接收人需在原定日期前归还。
